# Agent 指令

## 思考过程语言
所有内部思考、推理、分析过程必须使用简体中文。技术术语可保留英文，句子的主体必须是中文。

## 每次完成任务前
1. 读取 `.monkeycode/MEMORY.md`，逐条执行其中的规则
2. 读取本文件，遵守以下工作流程

## 工作流程
1. 当审计发现问题时，全部修复后再跑第二轮审计验证，确认零问题后再报告
2. 报告时只给结论，不列出中间过程
3. 每次改动后必须执行 `git add + commit + push`

## 代码质量标准

### 每次 Edit 后立即自查（不可跳过）
1. **确认编辑位置正确**：用 Read 工具读取修改处的上下 20 行，确认代码在正确的函数/分支/作用域内
2. **确认 oldString 完全匹配**：检查 Edit 的 oldString 是否精确匹配目标，未误删其他代码
3. **确认无残留代码**：检查被替换区域是否有遗漏的旧代码行
4. **花括号/圆括号平衡**：对所有修改过的文件跑 `python3 -c "检查括号"` 确认平衡

### 每次提交前全面审查
1. 全符号/命名/引用一致性检查
2. 检查函数调用链完整性
3. 检查 DOM id/class 引用与 HTML 定义一致
4. 检查变量命名新旧一致性
5. 检查所有路径覆盖（首次渲染/增量渲染/离线/在线）
6. 提交前运行 lint/语法验证

### 改完必读（不可跳过）
- 每处 Edit 后，必须 Read 修改位置的上下 20 行，确认：
  - 代码在正确的分支内（如 `if(existingCards)` 内才是增量路径）
  - 没有多余的 `}` 或缺少 `{`
  - 没有遗漏的旧代码行
  - 变量作用域正确
