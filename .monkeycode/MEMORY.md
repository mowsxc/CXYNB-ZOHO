提交前必须全文件符号与命名审查
- Date: 2026-07-01
- Context: 多次遗漏问题（函数调用但未定义、变量名不一致、DOM id 引用失效、增量渲染缺少组件），用户要求每次写完代码从 0 全面审查
- Instructions:
  - **每次提交前**，必须从 0 检查所有修改过的文件：
    - 扫描全部函数调用链，确认每个调用的函数都在作用域内已定义
    - 检查所有 DOM id/class 引用是否与 HTML 中定义的一致
    - 检查变量命名：新旧代码之间的变量名是否一致（如 `cardDefs` vs `cBackDefs` 要与 render 中的定义匹配）
    - 检查函数签名：调用时传入的参数个数和类型是否与定义一致
    - 检查模版字符串/HTML 拼接中的 `{`/`(` 是否闭合
    - 确认所有路径覆盖：首次渲染/增量渲染/离线/在线各分支都有对应处理
  - 审查顺序：HTML(DOM结构) → CSS(类名) → JS(函数定义→调用→变量) → 外部文件引用
  - 确认没有遗漏后，执行 git add + commit + push
